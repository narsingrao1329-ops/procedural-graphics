# Procedural Graphics (Code-as-Assets)

An exploration of generating graphics directly from source code.

Instead of relying on external assets, this project investigates how algorithms can generate visual content—from sprites and caves to infinite worlds.

---

## Philosophy

Traditional graphics pipelines:

```
assets → renderer
```

This project:

```
code → graphics
```

If it can be mathematically described, it can be procedurally generated.

## Inspiration

This project is inspired by the philosophy behind **TempleOS** and the work of **Terry A. Davis**, particularly the idea that graphics and assets can be represented directly in code.

The goal is not to recreate TempleOS, but to explore procedural graphics as a modern creative and educational experiment.
